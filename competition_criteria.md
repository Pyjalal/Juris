# KitaHack 2026 — Competition Criteria Reference

## CRITICAL TIMELINE
- **Submission Deadline: 28th February 2026** (5 days from Feb 23)
- Evaluation Period: 1st-7th March 2026
- Top 10 Announced: 9th March 2026
- Final Demo Day: 29th March 2026

## Submission Requirements
1. **Project Description** (Mandatory)
2. **Demo Video** (Max 5 minutes, YouTube unlisted/public, -1 mark per 30s over)
3. **Public Code Repository** (GitHub/GitLab with README + setup instructions)
4. **Project Documentation** (Tech overview, innovation, challenges)
5. **Submission Form Responses** (Google Forms)

## Technical Stack Requirements
- **MUST** use Google AI Technology (Gemini, Vertex AI, Google AI Studio)
- **MUST** use at least one additional Google Developer Technology (Firebase, GCP, Maps API, etc.)
- Non-Google AI models = **only half marks** for AI implementation
- No-code submissions NOT accepted
- Preliminary: minimum low-code, semi-working prototype encouraged
- Final: working prototype required

## Preliminary Round Scoring (80 points total)

### CATEGORY A: IMPACT (75% = 60 points)

| Points | Criteria | Details |
|--------|----------|---------|
| **15** | Problem Statement & SDG Alignment | Clear problem definition, strong SDG connection, AI solves the problem, well-justified SDG choice |
| **10** | User Feedback & Iteration | 3+ real user insights gathered, evidence of iteration based on feedback |
| **5** | Success Metrics & Scalability | Measurable goals, clear roadmap for scaling |
| **20** | AI Integration | AI used meaningfully, justified choice of Google AI, explained benefits |
| **10** | Technology Innovation | Creative use of tech for SDG, utilizes tech to solve real problems |

### CATEGORY B: TECHNOLOGY (25% = 20 points)

| Points | Criteria | Details |
|--------|----------|---------|
| **5** | Technical Architecture & Google Tech | Clear system architecture, justified Google Cloud/Firebase/Gemini usage |
| **5** | Technical Implementation & Challenges | All components implemented, highlighted major challenge + resolution |
| **10** | Completeness & Demonstration | Working prototype demonstrated, real coding preferred (Flutter, GitHub) |

## Final Round Scoring
- Problem Statement/Impact: 40%
- Presentation: 40%
- Technical: 20%

## Submission Form Structure
### Part 1: Team Details
- GDGoC member confirmation
- Google AI + Google Dev Tech confirmation
- Team name, leader details, member details

### Part 2: Project Details
- Project Name
- Google Technology used
- GitHub/Prototype Link
- Pitching video link (5 min)

### Part 3: Project Evaluation
- Challenge description
- SDG goals and targets with reasoning
- Solution architecture + component responsibilities
- Products/platforms chosen + reasons
- 3+ user feedback insights + iteration evidence
- Success metrics + scaling roadmap
